• gitのバージョン確認：git -v
  • ﾒｰﾙｱﾄﾞﾚｽが登録されているか確認：git config --global user.email
  • ユーザー名登録：git config --global user,name '名前'
  • ﾒｰﾙｱﾄﾞﾚｽ登録：git config --global user-email 'ﾒｰﾙｱﾄﾞﾚｽ'
  • ファイル修正後：git add
  • コミットメッセージ：git commit -m "ユニークメッセージ"
  • githubに送る：git push
  • init:initialize:初期化

  • 共同開発

  • 1,コードを変更する：git add

  • 2,共有する準備をする:git commit -m "コミットメッセージ"

  • 3,共有する:アップロード：git push

  • git remote add origin "URL":リモートリポジトリー登録

  • リモートリポジトリからファイルをダウンロード：git pull origin master

  • add→ commit→ push→ pull

  • リモートリポジトリにファイルをアップロードしたり、リモートリポジトリからファイルをダウンロードすることで、開発者同士がファイルを共有することが出来る
  • 完成しているコード→相手に共有
  • 未完成のコード→共有せず
  • git status:自分が変更したファイルのファイル名を表示可能
  • git diff:変更内容把握：変更前は赤、変更後は緑
  • git status:addされたのは赤、 addされていないのは緑
  • git log:自分や他人のコミット確認可能
  • git log -p：過去の変更内容確認：上下キーを使うことで表示範囲を変えられて、Qキーを押すと終了可能
