変数：データ（値）の入れ物（箱）。箱についている名前が変数名、箱の中に実際の値（文字列、数値）が入っている

let 変数名＝値

=：右辺を左辺に代入する

let ：これから変数を定義します

変数を使うと変数の部分は代入した値に置き換わる

なぜ変数を使うのか：１，同じ値は繰り返し使える２，変更に対応しやすい３，値の意味が分かり易い

命名ルール：1number：数字開始,bango：ローマ字,番号：日本語；不可

number：英単語,oddNumber：2単語以上の場合は大文字で区切る：OK

更新する場合：letが必要なくなり、変数名＝新しい値で良くなる

プログラムは上から順に実行されるため後で代入された値で変数の中身が変わる

  • x = x + 10:x +=10
  • x = x - 10:x -=10
  • x = x * 10:x *=10
  • x = x / 10:x /=10
  • x = x % 10:x %=10

変数は一度代入した値を更新できるが定数は値を更新出来ない

const = 定数

テンプレートリテラル：文字列の中で${定数}とすると、文字列の中に定数、変数を含めることが出来る

文字列全体をバッククォーテーション（'）で囲む必要がある。

複数の定数、変数を埋め込むことも可能

if文

ある条件が成り立つときだけある処理を行う＝条件分岐

インデントとは字下げのこと tabキーを押すと出来る console.logを書く前はインデントを行う

真偽値にはtrue・falseの2つの値しか存在しない。条件式は成り立つとtrue、成り立たなければfalseという真偽値に置き換わる

a < b:aはbより小さい

a <= b:aの方が小さい、または等しい

a > b:aはbより大きい

a >= b:aの方が大きい、または等しい

a == b:aとbが等しい

a != b:aとbが異なる

number == 12:true

number == "12":true:文字列を数値に変換して比較しているのでtrueになる

a === b:aとbが厳密に等しい

a !== b:aとbが厳密に異なる

else:もし～だったら～を行う、そうでなければ～を行う

if文の条件がfalseの場合、elseの処理が実行される

elseを使わない場合→複数のif文が必要

elseを使う場合→1つの条件式で成立する

else if(条件):if文に条件分岐を追加することが出来る

&&:かつ

条件1 && 条件2:条件1かつ条件2：複数の条件が全てtrueならtrueになる

||:または

条件1 || 条件2:条件1または条件2

複数の条件のうち1つでもtureならtureになる

switch{条件の値}{処理}文:ある値によって処理を分岐する

swich文の中にcaseを追加する事で処理を分けることが出来る

breakとはswitch文を終了する命令

breakがないと、合致したcaseの処理を行った後、その次のcaseの処理も実行される

default:caseのどれも一致しなかった時

while文:繰り返し処理：~の間：条件式がtrueの間、{}内の処理を繰り返す

条件式の判定をし、trueの場合{}の中の処理が一度実行。再び条件式がtrueか判断し、trueであればもう一度実行される。

条件式がいつかはfalseになり、繰り返し処理を終わらせる必要がある。いつまでもtrueの場合、永遠に繰り返し処理を実行する事となる：無限ループ

for文でも繰り返し処理を行うことが可能。

変数の定義・条件式・変数の更新の3つを()の中に書く、()の中ではそれぞれを;で区切る

number += 1:number ++

number -= 1:number --

複数の値をまとめて管理：配列

[値1,値2,値3]：配列に入っているそれぞれの値を要素という

配列も1つの値なので定数に代入可能

配列の要素にはそれぞれインデックス番号が存在し、０から始まる

配列[インデックス番号]

配列.length:配列の要素数を取得

配列の要素数が終わっても問題なく繰り返すことが出来る

オブジェクト：配列と同じく複数のデータをまとめて管理するもの

配列は複数の値を並べて管理するのに対し、オブジェクトはそれぞれの値にプロパティという名前を付けて管理する

配列は[]で囲み、オブジェクトは{}で囲む

プロパティと値の間は:で繋ぎ、プロパティ間は,で区切る

オブジェクトも定数に代入可能

オブジェクトの値を取り出すのは「オブジェクト.プロパティ名」

「オブジェクト.プロパティ名=新しい値」でオブジェクトの値を更新可能

配列の要素には文字列、数値、オブジェクトを使える：配列を作ることが可能

配列の中のオブジェクトのプロパティを取り出すには：「配列[インデックス番号].プロパティ名」

配列の存在しないインデックス番号やオブジェクトの存在しないプロパティの要素を取得：undefined:値が定義されていない

オブジェクトの値の部分は文字列、数値だけではなくオブジェクトも用いる

「オブジェクト名.プロパティ.プロパティ」を使う

1つのプロパティに複数の値を当てはめる時、配列も使うことが出来る

関数：いくつかの処理をまとめたもの

function()と書いて{}の中にまとめたい処理を書くことで関数を用意できる：関数を定義する

関数を定義した定数名を用いて「定数名」と書き関数の中の処理を実行する：関数を呼び出す

() =>:アロー関数

引数：関数に与える追加要素で一緒に値を渡すことで関数の中で値を利用可能

(引数名)=>

引数を受け取る関数を呼び出す「定数名(値)」：関数は指定した値を受け取り引数に代入する

複数の引数に値を渡して関数を呼び出すには,を使う、渡す引数の順番と関数を定義した引数の順番は対応している

呼び出し元で受け取る処理結果を戻り値という：関数が戻り値を返す

return:呼び出し元で値を受け取れる

「return 値」：関数は値を戻り値として返す

returnは戻り値を返すだけでなく、関数の処理を終了させることも可能、returnの後の関数の処理は実行されない

スコープ：定数や変数の使用できる範囲

関数の外側で定義した定数、変数はプログラムのどこでも使える

関数の{}内で定義した定数、変数は関数の内側でのみ使用可能

関数の内側、外側で同じ名前の変数を定義した場合それらは別のものとして扱われる

ifやswitch等の条件文、for・while等の繰り返し文は{}を使う構文でもスコープを作る

オブジェクトの値の部分に関数を用いる：「定数名.プロパティ名()」

インスタンスが生成された直後に実行される

インスタンス：オブジェクト：「インスタンス.プロパティ名」クラスの外で使用できる

constructorの後の()内に引数を入れるとコンストラクタの処理内で使用可能

コンストラクタに引数として値を渡すには「new クラス名()」の（）内に値を追加する

メソッド：インスタンスの動作：「メソッド名(){}」:{}の中に行いたい処理を記述する

「インスタンス.メソッド名()」：メソッドを呼び出し処理を実行する

「this.プロパティ名」：メソッド内でインスタンスの値を使用

「this.メソッド名()」：メソッド内で他のメソッドを呼び出す

ファイルを分割にした時のエラーはそれぞれのファイルを関連付けし、必要な値を渡すことで解決可能

「export default クラス名」：クラスをエクスポート（出力）し、他のファイルに渡すことが可能

「import クラス名　from "./ファイル名"」：他のファイルに定義されているクラスを使用する際はインポート（読込）：.jsは省略可能

「import　定数名 from "./ファイル名"」：インポート

デフォルトエクスポートは1ファイル1つの値のみ使える、複数の値をエクスポートしたい場合は名前付きエクスポートを使用する

「import{値の名前１,値の名前２}from"./ファイル名"」：1つのファイルから複数のエクスポートが可能
